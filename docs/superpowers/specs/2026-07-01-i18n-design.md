# i18n Design — Default English

**Goal:** Add `pkg/cais/i18n` with English as the default locale; migrate hardcoded Portuguese strings in the reference app and scaffolds.

**Approach:** Key-based catalog (stdlib only). `LOCALE` env var selects locale (`en` default, `pt` supported). No Accept-Language negotiation in v1.

## API

```go
// pkg/cais/i18n
const DefaultLocale = "en"

func NewCatalog(locale string) *Catalog
func DefaultCatalog() *Catalog

func (c *Catalog) T(key string, args ...any) string
func (c *Catalog) HTMLLang() string   // "en", "pt-BR"
func (c *Catalog) OGLocale() string   // "en_US", "pt_BR"
func (c *Catalog) Funcs() template.FuncMap // t, htmlLang, ogLocale
```

Missing keys return the key (visible in dev).

## Wiring

- `cais.Config.Locale` from `LOCALE` env (default `en`)
- `cais.NewRenderer(fsys, catalog)` — nil catalog uses `DefaultCatalog()`
- Partials get the same template funcs as pages
- Handlers receive `*i18n.Catalog` for validation/flash strings
- `meta.DefaultPreview` / `PreviewHTML` default locale → `en_US`

## Locale files

- `locales/en.go` — all keys in English
- `locales/pt.go` — Portuguese translations (opt-in via `LOCALE=pt`)

## Templates

Replace hardcoded copy with `{{ t "key" }}` and `{{ htmlLang }}` / `{{ ogLocale }}` in `base.html`.

## Out of scope

- Per-user locale, Accept-Language, pluralization, external JSON/YAML files, `cais g locale`.
